Initial Sync
?

How initial sync works:

  • Couchbase (SG / App Services / Edge): active_only=true excludes deleted & removed docs from the feed server-side.
  • CouchDB: No active_only support — the worker filters out deleted/removed docs in the processing layer.
  • include_docs=false — only _id, _rev, _seq are returned; full docs fetched via _bulk_get in batches.
  • feed=normal — one-shot request (not longpoll blocking).
  • A crash-safe initial_sync_done flag is saved in the checkpoint. If interrupted, the worker resumes initial sync from the last checkpoint.
